TaskRunner Overview

TaskRunner is a comprehensive task execution and management system for Laravel applications. It provides powerful features for running tasks on single or multiple servers with advanced monitoring, analytics, and recovery capabilities.

High Performance

Execute tasks efficiently with parallel processing, optimized resource usage, and intelligent caching.

Advanced Analytics

Comprehensive performance metrics, resource monitoring, and optimization insights.

Enterprise Security

Secure task execution with authentication, authorization, and audit logging.

Rollback & Recovery

Automatic rollback capabilities and recovery mechanisms for failed tasks.

Templates & Blueprints

Reusable task templates and blueprints for rapid development and deployment.

Multi-Server Support

Execute tasks across multiple servers with load balancing and failover support.

Key Benefits

  • Reduced development time with pre-built templates and blueprints
  • Improved reliability with automatic rollback and recovery
  • Enhanced monitoring and alerting for production environments
  • Scalable architecture for handling complex task workflows
  • Comprehensive analytics for performance optimization

Installation

TaskRunner is already included in your Laravel application. No additional installation is required.

Requirements

  • Laravel 10.x or higher
  • PHP 8.1 or higher
  • Database (MySQL, PostgreSQL, SQLite)
  • Queue system (Redis, Database, etc.)

Configuration

The TaskRunner module is automatically configured. You can customize settings by publishing the configuration file:

php artisan vendor:publish --tag=task-runner-config

This will create a config/task-runner.php file where you can customize:

  • • Default timeouts and retry settings
  • • Analytics and monitoring configuration
  • • Template and blueprint settings
  • • Callback and webhook configurations
  • • Rollback and recovery options

Basic Usage

Creating Your First Task

TaskRunner provides multiple ways to create and execute tasks. Here's the simplest approach:

use App\Modules\TaskRunner\AnonymousTask;

// Create a simple command task
$task = AnonymousTask::command('backup-database', 'mysqldump -u root -p myapp > backup.sql');

// Execute the task
$result = $task->dispatch();

if ($result->successful()) {
    echo "Backup completed successfully!";
} else {
    echo "Backup failed: " . $result->errorOutput();
}

For more complex tasks, you can create custom task classes:

use App\Modules\TaskRunner\BaseTask;

class DatabaseBackupTask extends BaseTask
{
    public function render(): string
    {
        return "mysqldump -u {$this->username} -p{$this->password} {$this->database} > {$this->backupPath}";
    }
}

// Usage
$backupTask = new DatabaseBackupTask();
$backupTask->setData('username', 'root');
$backupTask->setData('password', 'secret');
$backupTask->setData('database', 'myapp');
$backupTask->setData('backupPath', '/var/backups/myapp.sql');

$result = $backupTask->dispatch();

Task Execution Options

Synchronous Execution

// Execute and wait for completion
$result = $task->dispatch();

if ($result->successful()) {
    echo $result->output();
}

Background Execution

// Execute in background
$task->inBackground()->dispatch();

// Check status later
if ($task->isFinished()) {
    echo $task->getOutput();
}
