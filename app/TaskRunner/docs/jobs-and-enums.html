Jobs & Enums

Complete reference for TaskRunner queue jobs and enumeration types.

BackgroundTaskMonitorJob

Queued job that periodically monitors background tasks and triggers appropriate callbacks and status updates. 

use App\Modules\TaskRunner\Jobs\BackgroundTaskMonitorJob;
use App\Modules\TaskRunner\Models\Task;

// The job is automatically dispatched by BackgroundTaskTracker
// You can also dispatch it manually

// Dispatch immediately
BackgroundTaskMonitorJob::dispatch($taskId, $taskClass);

// Dispatch with delay
BackgroundTaskMonitorJob::dispatch($taskId, $taskClass)
    ->delay(now()->addSeconds(30));

// Dispatch to specific queue
BackgroundTaskMonitorJob::dispatch($taskId, $taskClass)
    ->onQueue('task-monitoring');

// Dispatch with retry configuration
BackgroundTaskMonitorJob::dispatch($taskId, $taskClass)
    ->tries(3)
    ->backoff(5);

// The job automatically:
// 1. Retrieves the task from database
// 2. Creates task instance
// 3. Calls BackgroundTaskTracker::monitorTask()
// 4. Schedules next monitoring if needed
// 5. Handles failures gracefully

Job Configuration

Queue Settings

  • Queue: task-monitoring
  • Max retries: 3
  • Backoff: 5 seconds
  • Timeout: 60 seconds

Monitoring Logic

  • Task status checking
  • Timeout detection
  • Progress calculation
  • Callback triggering

RetryCallbackJob

Handles retry logic for failed callback attempts with exponential backoff and failure tracking. 

use App\Modules\TaskRunner\Jobs\RetryCallbackJob;
use App\Modules\TaskRunner\Enums\CallbackType;

// This job is automatically dispatched by CallbackService
// when a callback fails

// Manual dispatch (if needed)
RetryCallbackJob::dispatch(
    task: $task,
    callbackType: CallbackType::Finished,
    additionalData: ['event' => 'task_completed'],
    attempt: 1
);

// Dispatch with custom retry configuration
RetryCallbackJob::dispatch(
    task: $task,
    callbackType: CallbackType::Failed,
    additionalData: ['error' => 'Connection timeout'],
    attempt: 2
)->delay(now()->addSeconds(30));

// The job automatically:
// 1. Attempts to send the callback
// 2. Handles success/failure
// 3. Schedules next retry if needed
// 4. Logs retry attempts
// 5. Gives up after max attempts

ProcessMonitoringAlertJob

Sends alerts for process monitoring events such as timeouts, failures, or performance issues. 

use App\Modules\TaskRunner\Jobs\ProcessMonitoringAlertJob;

// Dispatch alert job
ProcessMonitoringAlertJob::dispatch([
    'task_id' => $taskId,
    'alert_type' => 'timeout',
    'message' => 'Task exceeded timeout limit',
    'severity' => 'warning',
    'data' => [
        'timeout_limit' => 300,
        'actual_time' => 350,
        'task_name' => 'database-backup'
    ]
]);

// Dispatch with custom channels
ProcessMonitoringAlertJob::dispatch([
    'task_id' => $taskId,
    'alert_type' => 'memory_usage',
    'message' => 'High memory usage detected',
    'severity' => 'critical',
    'channels' => ['slack', 'email', 'webhook'],
    'data' => [
        'memory_usage' => 85,
        'threshold' => 80
    ]
]);

// The job automatically:
// 1. Formats alert message
// 2. Sends to configured channels
// 3. Handles delivery failures
// 4. Logs alert attempts
// 5. Tracks alert history

    'checkpoint_id' => $checkpointId,

    'task_id' => $taskId,

    'rollback_strategy' => 'partial_rollback',

// 1. Validates rollback parameters

TaskStatus Enum

Defines all possible states a task can be in during its lifecycle. 

use App\Modules\TaskRunner\Enums\TaskStatus;

// Available status values
TaskStatus::Pending;      // Task is waiting to be executed
TaskStatus::Running;      // Task is currently executing
TaskStatus::Finished;     // Task completed successfully
TaskStatus::Failed;       // Task failed during execution
TaskStatus::Timeout;      // Task timed out
TaskStatus::Cancelled;    // Task was cancelled
TaskStatus::UploadFailed; // Task file upload failed
TaskStatus::ConnectionFailed; // Task connection failed

// Helper methods
$status = TaskStatus::Running;

// Get description
echo $status->getDescription(); // "Task is currently executing"

// Get CSS class
echo $status->getCssClass(); // "status-running"

// Get icon
echo $status->getIcon(); // "🔄"

// Check if complete
if ($status->isComplete()) {
    echo "Task is finished";
}

// Check if active
if ($status->isActive()) {
    echo "Task is still running";
}

// Check if successful
if ($status->isSuccessful()) {
    echo "Task succeeded";
}

// Check if failed
if ($status->isFailed()) {
    echo "Task failed";
}

// Get all completed statuses
$completedStatuses = TaskStatus::getCompletedStatuses();

// Get all failed statuses
$failedStatuses = TaskStatus::getFailedStatuses();

Status Values

Active Statuses

  • Pending: ⏳ Waiting to execute
  • Running: 🔄 Currently executing

Completed Statuses

  • Finished: ✅ Successfully completed
  • Failed: ❌ Execution failed
  • Timeout: ⏰ Exceeded time limit
  • Cancelled: 🚫 Manually cancelled

CallbackType Enum

Defines all types of callbacks that can be sent during task execution. 

use App\Modules\TaskRunner\Enums\CallbackType;

// Available callback types
CallbackType::Custom;     // Custom callback
CallbackType::Timeout;    // Task timed out
CallbackType::Failed;     // Task failed
CallbackType::Finished;   // Task finished successfully
CallbackType::Started;    // Task started execution
CallbackType::Progress;   // Task progress update
CallbackType::Cancelled;  // Task was cancelled
CallbackType::Paused;     // Task was paused
CallbackType::Resumed;    // Task was resumed

// Helper methods
$callbackType = CallbackType::Finished;

// Get description
echo $callbackType->getDescription(); // "Task finished successfully"

// Get HTTP method
echo $callbackType->getHttpMethod(); // "POST"

// Check if completion callback
if ($callbackType->isCompletion()) {
    echo "This is a completion callback";
}

// Check if failure callback
if ($callbackType->isFailure()) {
    echo "This is a failure callback";
}

// Check if success callback
if ($callbackType->isSuccess()) {
    echo "This is a success callback";
}

// Check if lifecycle callback
if ($callbackType->isLifecycle()) {
    echo "This is a lifecycle callback";
}

// Get priority level
echo $callbackType->getPriority(); // 2 (medium priority)

// Priority levels:
// 1 = High priority (Failed, Timeout)
// 2 = Medium priority (Finished, Cancelled)
// 3 = Normal priority (Started, Progress)
// 4 = Low priority (Paused, Resumed, Custom)

Callback Categories

Lifecycle Callbacks

  • Started: Task begins execution
  • Paused: Task is paused
  • Resumed: Task is resumed
  • Cancelled: Task is cancelled

Status Callbacks

  • Progress: Progress update
  • Finished: Successful completion
  • Failed: Execution failure
  • Timeout: Time limit exceeded

InteractionType Enum

Defines types of user interactions that can be triggered during task execution. 

use App\Modules\TaskRunner\Enums\InteractionType;

// Available interaction types
InteractionType::Banner;    // Display banner notification
InteractionType::Dialog;    // Show dialog/modal
InteractionType::Toast;     // Show toast notification
InteractionType::Alert;     // Show alert box
InteractionType::Confirm;   // Show confirmation dialog
InteractionType::Prompt;    // Show input prompt

// Usage in tasks
$this->interact(InteractionType::Banner, [
    'title' => 'Task Started',
    'message' => 'Database backup has begun',
    'type' => 'info'
]);

$this->interact(InteractionType::Confirm, [
    'title' => 'Confirm Action',
    'message' => 'Are you sure you want to proceed?',
    'confirm_text' => 'Yes, proceed',
    'cancel_text' => 'Cancel'
]);

$this->interact(InteractionType::Dialog, [
    'title' => 'Task Progress',
    'content' => 'Task is 50% complete',
    'actions' => [
        ['label' => 'Continue', 'action' => 'continue'],
        ['label' => 'Cancel', 'action' => 'cancel']
    ]
]);

Interaction Features

Notification Types

  • Banner: Persistent banner
  • Toast: Temporary notification
  • Alert: Important message

Interactive Types

  • Dialog: Modal dialog
  • Confirm: Confirmation prompt
