Rollback & Recovery Overview

TaskRunner's rollback and recovery system provides comprehensive protection against task failures. It enables automatic and manual rollback capabilities with checkpoint-based recovery mechanisms.

Automatic Recovery

Automatically detect failures and rollback to the last known good state.

Checkpoint System

Create checkpoints at critical points to enable precise rollback.

Failure Detection

Intelligent failure detection with configurable thresholds and conditions.

Manual Rollback

Manual rollback capabilities for controlled recovery scenarios.

Rollback Analytics

Track rollback events and analyze recovery patterns for optimization.

Secure Rollback

Secure rollback operations with authentication and audit logging.

Checkpoint System

Creating Checkpoints

Checkpoints capture the current state of your system at critical points:

// Basic checkpoint creation
$task->createCheckpoint();

// Checkpoint with custom metadata
$task->createCheckpoint([
    'description' => 'Pre-deployment state',
    'version' => 'v2.1.0',
    'environment' => 'production',
    'backup_files' => ['config.php', 'database.sql']
]);

// Checkpoint with automatic rollback script
$task->setRollbackScript('
    echo "Rolling back to checkpoint..." &&
    git reset --hard HEAD~1 &&
    php artisan migrate:rollback &&
    echo "Rollback completed"
')->createCheckpoint();

Checkpoint Data

  • State: Current system state and configuration
  • Files: Backup of critical files and directories
  • Database: Database state and transaction logs
  • Metadata: Custom metadata and descriptions

Checkpoint Types

  • Full: Complete system state backup
  • Incremental: Changes since last checkpoint
  • Selective: Specific components only
  • Automatic: Created at predefined intervals

Checkpoint Management

Manage and restore checkpoints:

// Get checkpoint information
$checkpointInfo = $task->getRollbackInfo();

// List available checkpoints
$checkpoints = $task->listCheckpoints();

// Restore to specific checkpoint
$task->restoreCheckpoint('checkpoint_123');

// Delete old checkpoints
$task->cleanupCheckpoints([
    'keep_last' => 5,
    'older_than' => '7 days'
]);

// Validate checkpoint integrity
$isValid = $task->validateCheckpoint('checkpoint_123');

Basic Rollback Implementation

Simple Task with Rollback

class DatabaseMigrationTask extends BaseTask
{
    public function handle(): void
    {
        // Configure rollback
        $this->setRollbackScript('
            echo "Rolling back database migration..." &&
            php artisan migrate:rollback --step=1 &&
            echo "Database rollback completed"
        ')->setRollbackConfiguration([
            'enabled' => true,
            'automatic' => true,
            'on_failure' => true,
            'max_attempts' => 3
        ]);

        try {
            // Create checkpoint before migration
            $this->createCheckpoint([
                'description' => 'Pre-migration state',
                'migration_batch' => $this->getNextBatchNumber()
            ]);

            // Run migration
            $this->runCommand('php artisan migrate --force');

            // Verify migration success
            $this->runCommand('php artisan migrate:status');

        } catch (\Exception $e) {
            // Automatic rollback will be triggered
            $this->executeRollbackIfRequired('Migration failed: ' . $e->getMessage());
            throw $e;
        }
    }
}

Rollback Configuration

// Configure rollback behavior
$task->setRollbackConfiguration([
    'enabled' => true,
    'automatic' => true,
    'on_failure' => true,
    'on_timeout' => true,
    'max_attempts' => 3,
    'timeout' => 300,
    'notify_on_rollback' => true,
    'preserve_logs' => true,
    'rollback_script' => 'rollback.sh',
    'checkpoint_before_rollback' => true
]);

// Enable/disable rollback
$task->enableRollback();
$task->disableRollback();

// Get rollback information
$rollbackInfo = $task->getRollbackInfo();

Configuration Options

  • enabled: Enable rollback functionality
  • automatic: Automatic rollback on failure
  • on_failure: Rollback when task fails
  • on_timeout: Rollback on timeout
  • max_attempts: Maximum rollback attempts
  • timeout: Rollback operation timeout

Advanced Options

  • notify_on_rollback: Send notifications
  • preserve_logs: Keep rollback logs
  • rollback_script: Custom rollback script
  • checkpoint_before_rollback: Create checkpoint
  • validate_after_rollback: Verify rollback success

Advanced Rollback Features

Conditional Rollback

Rollback based on specific conditions:

// Conditional rollback based on exit code
if ($exitCode !== 0) {
    $this->executeRollbackIfRequired('Non-zero exit code: ' . $exitCode);
}

// Conditional rollback based on output
$output = $this->runCommand('php artisan health:check');
if (str_contains($output, 'ERROR')) {
    $this->executeRollbackIfRequired('Health check failed');
}

// Conditional rollback based on time
$startTime = time();
// ... perform operations ...
if (time() - $startTime > 300) { // 5 minutes
    $this->executeRollbackIfRequired('Operation timed out');
}

// Conditional rollback based on custom criteria
if ($this->shouldRollback()) {
    $this->executeRollbackIfRequired('Custom rollback condition met');
}

Scheduled Rollback

Schedule rollback operations for later execution:

// Schedule rollback if required
$this->scheduleRollbackIfRequired('Scheduled rollback reason');

// Schedule rollback with delay
$this->scheduleRollbackIfRequired('Delayed rollback', [
    'delay' => 300, // 5 minutes
    'reason' => 'Grace period expired'
]);

// Schedule rollback with conditions
$this->scheduleRollbackIfRequired('Conditional rollback', [
    'conditions' => [
        'health_check_fails' => true,
        'performance_degraded' => true
    ],
    'delay' => 600 // 10 minutes
]);

Rollback Chains

Execute multiple rollback operations in sequence:

Real-World Examples

Database Migration Rollback

Complete database migration with rollback protection:

class DatabaseMigrationTask extends BaseTask
{
    public function handle(): void
    {
        $migrationBatch = $this->getNextBatchNumber();
        
        // Configure rollback
        $this->setRollbackScript("
            echo 'Rolling back database migration...' &&
            php artisan migrate:rollback --batch={$migrationBatch} &&
            echo 'Database rollback completed'
        ")->setRollbackConfiguration([
            'enabled' => true,
            'automatic' => true,
            'on_failure' => true,
            'max_attempts' => 3,
            'timeout' => 600
        ]);

        try {
            // Create checkpoint
            $this->createCheckpoint([
                'description' => "Pre-migration batch {$migrationBatch}",
                'migration_batch' => $migrationBatch,
                'database_backup' => $this->createDatabaseBackup()
            ]);

            // Run migrations
            $this->runCommand("php artisan migrate --force");
            
            // Verify migrations
            $this->runCommand("php artisan migrate:status");
            
            // Test database connectivity
            $this->runCommand("php artisan db:monitor");

        } catch (\Exception $e) {
            $this->executeRollbackIfRequired("Migration failed: {$e->getMessage()}");
            throw $e;
        }
    }

    private function createDatabaseBackup(): string
    {
        $backupFile = "backup_" . date('Y-m-d_H-i-s') . ".sql";
        $this->runCommand("mysqldump -u root -p database > {$backupFile}");
        return $backupFile;
    }
}

Deployment Rollback

Production deployment with comprehensive rollback:

class ProductionDeploymentTask extends BaseTask
{
    public function handle(): void
    {
        $version = $this->options['version'] ?? 'latest';
        $currentVersion = $this->getCurrentVersion();

        // Configure rollback
        $this->setRollbackScript("
            echo 'Rolling back deployment...' &&
            
            # Restore previous version
            git reset --hard {$currentVersion} &&
            
            # Restore configuration
            cp config.production.backup.php config.php &&
            
            # Rollback database if needed
            php artisan migrate:rollback --step=1 &&
            
            # Clear caches
            php artisan cache:clear &&
            php artisan config:clear &&
            
            # Restart services
            systemctl restart nginx &&
            systemctl restart php-fpm &&
            
            echo 'Deployment rollback completed'
        ")->setRollbackConfiguration([
            'enabled' => true,
            'automatic' => true,
            'on_failure' => true,
            'on_timeout' => true,
            'max_attempts' => 5,
            'timeout' => 900
        ]);

        try {
            // Create comprehensive checkpoint
            $this->createCheckpoint([
                'description' => "Pre-deployment version {$currentVersion}",
                'current_version' => $currentVersion,
                'target_version' => $version,
                'backup_files' => [
                    'config.php',
                    'composer.lock',
                    'package-lock.json'
                ],
                'database_backup' => $this->createDatabaseBackup()
            ]);

            // Deploy new version
            $this->runCommand("git pull origin main");
            $this->runCommand("composer install --no-dev --optimize-autoloader");
            $this->runCommand("php artisan migrate --force");
            $this->runCommand("php artisan cache:clear");
            $this->runCommand("php artisan config:clear");

            // Health check
            $healthCheck = $this->runCommand("php artisan health:check");
            if (str_contains($healthCheck, 'ERROR')) {
                throw new \Exception('Health check failed after deployment');
            }

        } catch (\Exception $e) {
            $this->executeRollbackIfRequired("Deployment failed: {$e->getMessage()}");
            throw $e;
        }
    }
}
